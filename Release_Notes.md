## Overview  

This release completes the **Customer Lot from Hold Inventory** feature by implementing the missing **Mets Hold Pre‑Receipt** components required by TrakSYS.

The original CR 187 enabled allocation of By‑Product splits in **METS HOLD** to a new lot and sent a bulk request to TrakSYS. However, TrakSYS could not process the bulk request because it was not aware of the newly created **master lot** generated by CAI.

This release resolves that gap by introducing a **PreReceipt message** flow that informs TrakSYS of the new master lot before the bulk request is processed.

## What’s New  

### Functional Enhancements  

This release introduces the following capabilities:

1. **Mets Hold Pre‑Receipt Mapping**  
   - New table and list page to define vendor‑contract‑specific data required for the PreReceipt message payload.

2. **Automated PreReceipt Messaging**  
   - A new API procedure sends a PreReceipt message to TrakSYS when a customer lot is created from hold inventory.

3. **New APIM Trigger Code**  
   - Added a dedicated trigger code to support Mets Hold Pre‑Receipt processing.

4. **New Lot Estimates**  
   - New table and page to configure estimated assay percentages per metal type at the vendor contract level.

5. **Improved Navigation**  
   - Direct navigation from the **Vendor Contract Card** to:
     - Mets Hold Pre‑Receipt Mapping
     - New Lot Estimates

## Setup Requirements  

### Mets Hold Pre‑Receipt Mapping  

To enable the feature, populate the table **JM.Mets Hold PreRcpt Map (50216)** for each vendor contract that uses the Mets Hold Pre‑Receipt feature.

Each mapping defines:

- **Work Paper Type** (Melt / Sample)
- **Evaluation Route Path Item**  
  *(Linked to an Item with `CAI Production BOM = true`)*
- **Customer Reference** (free text)

**Access Path:**  
Vendor Contract Card → **Navigation** → *Mets Hold Pre‑Receipt Mapping*  

> The page opens pre‑filtered to the selected vendor and contract.

---

### New Lot Estimates  

Estimated assay percentages can be configured per vendor contract and metal type.

**Access Path:**  
Vendor Contract Card → **Navigation** → *New Lot Estimates*

## Use Case: Creating a Customer Lot from Hold Inventory  

### Trigger Point  

The PreReceipt message is triggered from **CAI Inventory Ledger (Page 37031323)** when the user selects **Create New Lot**.

### Process Flow  

1. User navigates to **METS HOLD** inventory via the CAI Inventory Ledger page.
2. User selects production orders for the same customer and contract.
3. User selects **Create New Lot** with:
   - **Lot Type:** *Customer Lot Received at JM*
4. CAI:
   - Creates the lot with status **30 – Receipt Complete**
   - Sends a **PreReceipt message** to TrakSYS via middleware.
5. The PreReceipt message includes an optional **LotID** identifying the newly created master lot.
6. TrakSYS:
   - Creates the lot record
   - Progresses it to **RN Generated** status.
7. TrakSYS processes the subsequent bulk request from CAI.
8. TrakSYS sends updated container IDs back to CAI.

## PreReceipt Payload – Data Sources  

The PreReceipt message payload is assembled from the following sources:

| Data Source | Fields Populated |
|------------|----------------|
| **Vendor Contract** | ProductionRoute, SCMaterialTypeCode, MaterialDescription, COMAHCode, WasteMaterialClassCode, WasteMaterialTypeCode, MSDSNumber |
| **New Master Lot** | AdvisedWeightUOM, AdvisedWeightNett, AdvisedWeightGross, NumberOfContainers, LotID |
| **Mets Hold Pre‑Receipt Mapping** | WorkPaperType, EvaluationRoute, CustomerReference |

### Additional Sources  

- **Vendor:** Name, CustomerID, CustomerName  
- **General Ledger Setup:** EstimatedValueCurrency  
- **Material‑to‑Waste Mapping:** COMAHCode, WasteMaterialClassCode, WasteMaterialTypeCode, MSDSNumber
